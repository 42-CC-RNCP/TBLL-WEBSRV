# ‚úÖ Test Plan: RFC9112 ¬ß2.1 Request-Line Format

## üîπ Test Objective

To verify that the HTTP/1.1 server correctly handles valid and invalid request-line formats as specified by RFC9112 ¬ß2.1:

```
request-line = method SP request-target SP HTTP-version CRLF
```

Server must return `400 Bad Request` for any syntactically invalid request-line. Valid requests should be accepted without error.
